# WorldWriters

A multiplayer role-playing game client built with Godot Engine, featuring character creation, inventory management, group interactions, and immersive storytelling mechanics.

## Description

WorldWriters is a digital tabletop RPG client designed to facilitate collaborative storytelling and character management. The game includes:

- **Character Creation & Management**: Create and customize your character with detailed attributes, disciplines, and backgrounds
- **Multiplayer Support**: Connect to a game server and play with other players in real-time
- **Inventory System**: Manage your character's possessions and equipment
- **Group Mechanics**: Form groups, participate in vaulderie ceremonies, and manage blood bonds
- **Nightly Activities**: Track and manage your character's actions during game sessions
- **Dice Rolling System**: Built-in dice roller for resolving actions
- **Storyteller Tools**: Special interfaces for game masters to manage healing, virtue tests, and more

## Contributing

We welcome contributions from the community! Whether you're fixing bugs, adding features, or improving documentation, your help is appreciated.

### Getting Started

1. **Fork the Repository**
   - Click the "Fork" button at the top right of this repository
   - Clone your fork to your local machine:
     ```bash
     git clone https://github.com/YOUR_USERNAME/core.git
     cd core
     ```

2. **Set Up Development Environment**
   - Download and install [Godot Engine 4.4](https://godotengine.org/download)
   - Open the project in Godot by selecting the `project.godot` file
   - Ensure all dependencies are loaded correctly

3. **Create a Branch**
   - Create a new branch for your feature or bugfix:
     ```bash
     git checkout -b feature/your-feature-name
     ```
   - Use descriptive branch names (e.g., `fix/character-creation-bug`, `feature/new-dice-roller`)

### Making Changes

1. **Code Style**
   - Follow GDScript best practices and conventions
   - Use meaningful variable and function names
   - Comment complex logic and non-obvious code
   - Keep functions focused and modular

2. **Testing**
   - Test your changes thoroughly before submitting
   - Ensure existing functionality isn't broken
   - Test multiplayer features in both client and server environments

3. **Commit Your Changes**
   - Write clear, concise commit messages:
     ```bash
     git add .
     git commit -m "Add: Brief description of your changes"
     ```
   - Use prefixes like `Add:`, `Fix:`, `Update:`, `Remove:` for clarity

### Submitting a Pull Request

1. **Push to Your Fork**
   ```bash
   git push origin feature/your-feature-name
   ```

2. **Open a Pull Request**
   - Go to the [original repository](https://github.com/NorthernNights20/core)
   - Click "New Pull Request"
   - Select your fork and branch
   - Provide a clear title and description of your changes
   - Reference any related issues (e.g., "Fixes #123")

3. **Code Review**
   - Wait for maintainers to review your PR
   - Address any feedback or requested changes
   - Once approved, your changes will be merged!

### Areas to Contribute

- **Bug Fixes**: Check the Issues page for known bugs
- **New Features**: Propose and implement new game mechanics or UI improvements
- **Documentation**: Improve code comments, README, or user guides
- **Assets**: Contribute new themes, fonts, or visual elements
- **Performance**: Optimize existing code and reduce memory usage
